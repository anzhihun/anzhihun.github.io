OpenLayers 3 Primer

结合API文档分析代码

现在我们可以借助API文档来帮助我们分析之前那个地图的代码：

        // 新建一个地图
      new ol.Map({
            // 定义地图显示图层集合
            layers: [
              // 新建一个使用Open Street Map地图源的瓦片地图图层
              new ol.layer.Tile({source: new ol.source.OSM()})
            ],
            // 新建显示地图的视图
            view: new ol.View({
              center: [0, 0],    // 定义地图显示中心于经度0度，纬度0度处
              zoom: 2            // 并且定义地图显示层级为级别2
            }),
            // 设置id为map的dom元素来作为地图的容器
            target: 'map'    
        });

创建地图就是新建一个ol.Map类的对象，那么它带有什么构造参数，就可以通过API文档查看，你会发现其实除了上面代码中设置的参数外，还有很多其他的参数。通过阅读参数的描述，可以发现其他没设置的参数都是允许不设置的，在不设置的情况下，会自动设置一些默认值。通过这些参数你还会发现，它们其实就代表了所有地图的组成部分。虽然没有设置，但是默认值也足以让地图具有放大缩小按钮，可以进行鼠标拖拽。认真查看API文档，你会在这里发现很多东西，学习到很多东西。所以请重视API文档。

接下来我们看代码中ol.Map构造参数里面的layers，可以是ol.Collection，也可以是Javascript数组，此处用了数组。但里面只包含了一个layer，就是ol.layer.Tile的实例，从API文档上看，数组里面的元素应该是ol.layer.Base类型才是，为什么可以用ol.layer.Tile呢？这时，通过API文档中的继承关系信息，我们可以了解到ol.layer.Tile继承于ol.layer.Layer，而ol.layer.Layer又继承于ol.layer.Base，所以这是可行的。

那么ol.layer.Tile又是什么呢，通过API文档，你能看到相关的介绍说明，在此之前，我们通过包名，就可以推断出它是一个layer，属于地图层的一种。但具体代表什么，官网API文档说明：它是一种应用瓦片图片作为地图源的地图层。构造函数的参数里包含一个必须设置值的属性：source，用于设置地图源，类型为ol.source.Tile，但是我们的代码里面使用的是ol.source.OSM，为什么可以呢？这个问题留给你自行去解答吧。

在此我们可以了解到，其实layer是为source服务的，每一个layer都必须要有source，且source起决定因素，需要配套使用。为此在应用开发时，不知道应该使用哪一种layer时，我们可以先看一下有什么source可用。如何查找呢？在官网API中输入ol.source即可查看到现有哪些不同类型的source，然后挨个查看，直到找到需要的source类。假设找到了ol.source.Test，那么要找layer就简单了，一般情况下，会有ol.layer.Test。 如果没有，请查找ol.source.Test的父类对应的layer，以此类推。

参数View是一个必须设置的属性，对应于ol.View类，通过官网API了解详细信息，顺便详细看看它的构造参数中的center和zoom表示什么？View至关重要，后续将更深入的了解。

至此，通过上面的这段代码的分析，应该有了更深入的了解。不妨试试修改一些参数的值，查看地图有什么变化。
